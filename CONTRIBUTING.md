# Contributing to ChatGPT Archive Reader

Thank you for your interest in contributing to ChatGPT Archive Reader! This document provides guidelines and information for contributors.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/chatgpt-archive-reader.git`
3. Install dependencies: `npm install`
4. Start development: `npm start`

## Development Workflow

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test your changes thoroughly
4. Commit with clear messages: `git commit -m "Add: brief description of changes"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Create a Pull Request

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing naming conventions
- Add comments for complex logic
- Test your changes across different screen sizes

## Building for Distribution

To build the app for your platform:
```bash
npm run build:mac    # macOS
npm run build:win    # Windows
npm run build:linux  # Linux
```

## Reporting Issues

When reporting bugs, please include:
- Operating system and version
- Steps to reproduce the issue
- Expected vs actual behavior
- Any error messages or screenshots

## Feature Requests

Feature requests are welcome! Please:
- Check if the feature already exists or is planned
- Describe the use case and why it's needed
- Consider how it fits with the app's current design
